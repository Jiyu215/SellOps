# 주문 상세페이지 설계

> 화면 구성: 좌우 2단 / 상단 주요 정보 / 하단 상세 영역  
> 가로 기준 12칸
---

## 1. 상단 기본 정보 (12칸 전체, 높이 2~3행)
- 위치: 페이지 상단
- 차지 칸: 전체 12칸
- 내용:
  - 주문번호
  - 주문 일시
  - 주문 상태
  - 결제 상태
  - 결제 수단
- 설명: 운영자가 주문 핵심 정보를 바로 확인 가능

---

## 2. 고객/배송 정보 (좌우 2단, 각 6칸, 높이 3~4행)
- 위치: 상단 바로 아래
- 차지 칸:
  - 좌측 6칸 → 고객 정보
    - 이름, 연락처, 이메일, 청구 주소
  - 우측 6칸 → 배송 정보
    - 배송지 주소, 수령인 이름/연락처, 배송 상태, 송장관리(택배사/송장번호)
- 설명: 고객/배송 정보를 한눈에 비교 가능

---

## 3. 상품/결제/쿠폰 정보 (12칸, 높이 4~5행)
- 위치: 고객/배송 정보 아래
- 차지 칸: 전체 12칸
- 내용:
  - 상품 목록 테이블
    - 항목: 상품명 / 옵션 / 수량 / 단가
  - 총 합계 금액
  - 배송비
  - **쿠폰/할인 정보**
    - 적용 쿠폰 코드
    - 할인 금액
    - 쿠폰/프로모션 유형 (정액, 정률, 무료배송)
    - 사용 여부 (사용완료, 미사용, 취소)
- 설명: 상품/결제 정보와 함께 할인 적용 여부 확인

---

## 4. 주문 메모 (12칸, 높이 2~3행)
- 위치: 상품/결제/쿠폰 정보 아래
- 차지 칸: 전체 12칸
- 내용:
  - 고객 요청사항
  - 관리자 메모
- 설명: 운영자 참고용, 중요 사항 강조

---

## 5. 액션 버튼 / 상태 변경 (12칸, 높이 1~2행)
- 위치: 페이지 하단, 메모 바로 아래
- 차지 칸: 전체 12칸
- 내용: 상태 기반 버튼 + 보조 [⋯] 메뉴
- 상세 규칙:
  - 주문대기 → [주문확정, 주문취소]
  - 주문확정 → [배송시작] + [⋯ 메뉴: 주문취소, 상세보기]
  - 배송중 → [배송완료] + [⋯ 메뉴: 반품접수, 배송보류, 상세보기]
    - 배송보류(shipping_on_hold): 배송 보류 상태에 대한 정의는 존재하나, 현재 MVP에서는 해당 상태 및 기능을 구현하지 않는다.
  - 배송완료 → [환불처리] + [⋯ 메뉴: 상세보기]
  - 주문취소 / 환불완료 / 반품완료 → 버튼 없음
- 설명: 클릭 시 모달로 입력 받고 상태 전환, 로그 기록

---

## 6. 동작 흐름
1. 버튼 클릭 → 모달(입력/확인)  
2. 입력 완료 → 상태 전환 API 호출  
3. 상태 변경 → UI 갱신  
4. 모든 변경 사항 로그 기록

---

## 7. UI/UX 규칙
- 주요 정보는 페이지 상단에 노출  
- 고객/배송 정보는 좌우 2단 배치  
- 상품/결제/쿠폰 정보, 주문 메모는 전체 너비 12칸  
- 액션 버튼은 하단 고정, 상태 기반 동적 노출  
- 입력 모달을 통해 필요한 정보 수집 후 상태 변경  
- 쿠폰 정보는 읽기 전용, 총 결제금액에 할인 반영 표시

---

## 8. 반응형 레이아웃 규칙
### PC (≥1280px) — 기본 레이아웃
상단 기본 정보: 12칸 전체, Flex 또는 CSS Grid 1~2행
고객/배송 정보: CSS Grid 2열 (좌측 고객, 우측 배송)
상품/결제/쿠폰 정보: 12칸 전체, 테이블 Grid
주문 메모: 12칸 전체
액션 버튼: Flex row, 버튼 오른쪽에 ⋯ 메뉴
모달/입력창: position: fixed; 중앙 정렬

### 태블릿 (768px ~ 1279px)
상단 기본 정보: Flex wrap 또는 Grid 2행
고객/배송 정보: Grid → 1열씩 세로 배치
상품/결제/쿠폰 정보: 12칸, 테이블 스크롤 가능
주문 메모: 12칸
액션 버튼: Flex column, ⋯ 메뉴 아래
모달/입력창: 화면 중앙, 너비 축소

### 모바일 (≤767px)
상단 기본 정보: 각 항목 블록형 세로 정렬
고객/배송 정보: 세로 블록
상품/결제/쿠폰 정보: 카드형 레이아웃
카드 헤더: 주문번호 + 상태 뱃지
카드 내용: 상품명, 옵션, 수량, 단가, 총합, 배송비, 쿠폰 정보 → 2열 Grid
주문 메모: 세로 블록
액션 버튼: Flex column, ⋯ 메뉴 아래
모달/입력창: 화면 전체 너비 또는 중앙 고정
