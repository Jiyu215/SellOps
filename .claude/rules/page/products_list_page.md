# 상품 관리(목록) 페이지 

## 0. 페이지 개요
- 경로 : `/products`
- 목적 : 전체 상품의 현황 파악, 빠른 탐색·필터·일괄 작업
- 주요 사용자 : 상품 MD, 운영 담당자
- 진입 방식 : 사이드바 '재고관리' 메뉴 클릭 > 상품 관리 페이지 이동

---

## 1. 레이아웃 구조
```
┌─────────────────────────────────────────────────────────┐
│  PageHeader                                             │
│  [상품관리]          [CSV Export]  [+ 상품 등록]          │
├─────────────────────────────────────────────────────────┤
│  StatSummaryBar                                         │
│  전체 N건 | 판매중 N | 숨김 N | 품절 N                     │
├─────────────────────────────────────────────────────────┤
│  FilterBar                                              │
│  [검색창──────────]  [상태▼]  [정렬▼]  [초기화]            │
├─────────────────────────────────────────────────────────┤
│  BulkActionBar (행 선택 시 노출)                          │
│  N개 선택됨 | [상태변경▼]  [삭제]                          │
├─────────────────────────────────────────────────────────┤
│  ProductTable                                           │
│  ☐ | 이미지 | 상품명·코드 | 가격 | 재고 | 상태 | 수정일      │
│  ─────────────────────────────────────────────────────  │
│  row × N                                                │
├─────────────────────────────────────────────────────────┤
│  Pagination                                             │
│  [페이지당 N개▼]    < 1 2 3 … >    총 N건                 │
└─────────────────────────────────────────────────────────┘
```

## 2. PageHeader

### 구성 요소
- 타이틀 : 상품관리
- 보조 텍스트 : 현재 필터 상태 요약 ( 예: `전체 상품 · 243건` )
- 액션 버튼 영역 ( 우측 정렬 )

### 액션 버튼
- `CSV Export` : 현재 필터 조건 반영한 CSV 다운로드
- `+ 상품 등록` : `/products/new` 라우팅

#### CSV Export 상세
- 클릭 시 로딩 스피너 + 비활성화 처리 (중복 요청 방지)
- 다운로드 파일명 : `products_YYYYMMDD_HHmmss.csv`
- 포함 컬럼: 상품코드, 상품명, 판매가, 전체재고, 판매수량, 가용재고, 상태, 등록일, 수정일
- 현재 검색·필터 조건이 적용된 데이터 기준 (전체 페이지, 페이지네이션 무관)
- 성공 시 토스트 : `CSV 파일이 다운로드되었습니다.`
- 실패 시 토스트 : `다운로드에 실패했습니다. 다시 시도해주세요.`

## 3. StatSummaryBar
- 페이지 상단 집계 바
- 클릭 시 해당 상태로 필터 즉시 적용
- hover 시 Tooltip으로 원본 값 제공 ( 1.2K (hover → 1,234) )
- grid 기반 레이아웃으로 항목 수와 관계없이 안정적으로 정렬 (숫자 길이에 영향을 받지 않도록 grid layout 사용)

```
┌──────────┬──────────┬──────────┬──────────┐
│ 전체 1.2K │ 판매중198 │  숨김 32 │  품절 13  │
└──────────┴──────────┴──────────┴──────────┘
```

**데이터가 커질경우**
- 기본원칙
    - 1,000 이상 : K / M 단위로 축약
    - 1,000 미만 : 원본 숫자 그대로 표시
    - 축약 시 소수점 1자리 + 반올림 적용
- 예시
    - 1,200 → 1.2K
    - 12,300 → 12.3K
    - 1,200,000 → 1.2M

## 4. FilterBar

### 검색창
- Placeholder : `상품명 또는 상품코드 검색`
- 검색 방식 : 입력 시 debounce(300ms) 기반 자동 검색 + Enter / 아이콘 클릭 지원
- Debounce : 300ms
- 대상 필드 : `name` LIKE, `productCode` LIKE
- 한글 입력 : IME 조합 입력 처리 (조합 중 검색 지연, 완료 시 검색 실행)
- 공백 처리 : trim 후 검색
- 상태 유지 : 검색어는 URL query에 반영되어 새로고침 시 유지
- 초기화 : 검색어 입력 시 우측 X 버튼 노출

### 상태 필터 (Dropdown)
- 전체 : 전체 상태
- `active` : 판매중
- `hidden` : 숨김
- `sold_out` : 품절

- 기본값: 전체
- 선택 시 즉시 목록 갱신

### 정렬 (Dropdown)
- `createdAt_desc` : 최신 등록순 (기본)
- `createdAt_asc` : 오래된 등록순 
- `updatedAt_desc` : 최근 수정순 
- `name_asc` : 상품명 오름차순 
- `name_desc` : 상품명 내림차순 
- `price_asc` : 낮은 가격순 
- `price_desc` : 높은 가격순 
- `available_asc` : 재고 적은 순 
- `available_desc` : 재고 많은 순 

### 초기화 버튼
- 검색어, 상태 필터, 정렬 모두 초기값으로 리셋 
- 필터 적용 중일 때만 활성화

### URL 쿼리 파라미터 동기화
모든 필터·정렬·페이지 상태는 URL에 반영되어야 함

```
/products?search=나이키&status=active&sort=price_asc&page=2&limit=20
```
- 브라우저 뒤로가기 시 필터 상태 복원
- 링크 공유 시 동일 필터 상태 재현

## 5. BulkActionBar (일괄 작업 바)
행 체크박스 1개 이상 선택 시 FilterBar를 대체하여 노출
```
┌────────────────────────────────────────────────────────┐
│  ☑ 5개 선택됨    [상태 변경 ▼]  [선택 삭제]  [선택 취소] │
└────────────────────────────────────────────────────────┘
```

### 상태 변경 드롭다운
- 판매중으로 변경 : 선택 상품 `status → active`
- 숨김으로 변경 : 선택 상품 `status → hidden`
- 품절로 변경 : 선택 상품 `status → sold_out`

- 적용 전 확인 모달: `N개 상품의 상태를 [상태명]으로 변경하시겠습니까?`
- 성공 시 목록 즉시 갱신 + 토스트

### 선택 삭제
- 확인 모달: `선택한 N개 상품을 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.`
- 삭제 중 로딩 처리
- 성공 시 목록 갱신 + 선택 초기화

## 6. ProductTable
- ☐ (체크박스) : 전체 선택/해제 (헤더 체크박스)
- 이미지 : `type='list'` 이미지, 없으면 기본 이미지 placeholder
- 상품명 / 코드 : 상품명 (bold) + 상품코드 (sub text, gray)
- 판매가 : `₩ 1,000` 형식, 천단위 콤마
- 재고 : `available / total` 형식
- 상태 : 상태 Badge
- 수정일 : `YYYY.MM.DD`
- 관리 : `수정` 버튼 or 아이콘

### 재고 표시 규칙
- `available >= 10` : 기본
- `1 <= available <= 9` : 주의
- `available === 0` : 위험

### 행(Row) 인터랙션
- 상품명/코드 텍스트 클릭 : `/products/:id` 상세 이동
- `수정` 버튼 클릭 : `/products/:id` 상세 이동
- 체크박스 클릭 : 해당 행 선택/해제
- 헤더 체크박스 클릭 : 현재 페이지 전체 선택/해제

### 빈 상태 (Empty State)
```
       [이미지 아이콘]
   검색 결과가 없습니다.
  조건을 변경하여 다시 검색해보세요.
      [필터 초기화]
```

- 필터 없이 데이터 자체가 없는 경우:
```
       [상자 아이콘]
   등록된 상품이 없습니다.
  첫 번째 상품을 등록해보세요.
      [+ 상품 등록]
```

### 로딩 상태
- 테이블 전체 영역에 Skeleton UI
- 페이지 전환·필터 변경 시 기존 데이터 유지 + 로딩 오버레이 (UX 안정성)

## 7. Pagination
```
[20개▼]    [이전] 1 2 3 … 12 [다음]    총 243건
```
- 페이지당 항목 수 : 10 / 20 / 50 / 100 선택 가능, 기본 20
- 페이지 번호 : 현재 페이지 강조, 최대 5개 번호 표시 + 줄임표
- 총 건수 : 현재 필터 기준 총 건수 표시
- URL 반영 : `?page=2&limit=20`

---

## 8. 성능 요구사항
- 목록 최초 로딩 : < 1.5초 (LCP 기준)
- 필터·정렬 갱신 : < 800ms
- CSV Export (1,000건) : < 3초
- 이미지 썸네일 : Lazy Loading 적용
- API 요청 : 페이지네이션·필터 조건 서버사이드 처리

## 9. 반응형 (Responsive) 정의
- PC (≥1280px) — 기본 레이아웃 : 전체 컬럼 표시
- 1024px ~ 1279px : `수정일` 컬럼 숨김
- 태블릿 (768px ~ 1279px) : `수정일`, `재고` 컬럼 숨김, 이미지 축소
- 모바일 (≤767px) : 테이블 → 카드 리스트 전환

### 모바일 카드 레이아웃
```
┌───────────────────────────────┐
│ [썸네일] 상품명                 │
│         상품코드               │
│         ₩19,800  [판매중]      │
│         재고: 45개             │
└───────────────────────────────┘
```

## 10. 접근성 (Accessibility)
- 테이블 `<caption>` 제공
- 체크박스 `aria-label` 설정 (예: `상품명 선택`)
- 정렬 버튼 `aria-sort` 속성 관리
- 상태 Badge `aria-label` (색상만으로 의미 전달 금지)
- 키보드 탐색: Tab 이동, Enter/Space 선택 지원
- 포커스 링 스타일 명시

---

## 11. 에러 처리
- 목록 조회 실패 : 에러 메시지 + `다시 시도` 버튼
- 일괄 상태 변경 실패 : 에러 토스트 + 선택 유지
- CSV Export 실패 : 에러 토스트
- 네트워크 오류 : 오프라인 배너 + 재시도 유도

## 12. API 연동 정의
 
### 목록 조회
```
GET /api/products
Query: {
  search?: string,
  status?: 'active' | 'hidden' | 'sold_out',
  sort?: string,       // e.g. 'price_asc'
  page: number,        // 1-based
  limit: number        // 10 | 20 | 50 | 100
}
Response: {
  items: ProductListItem[],
  total: number,
  page: number,
  limit: number,
  summary: {           // StatSummaryBar 용
    total: number,
    active: number,
    hidden: number,
    sold_out: number
  }
}
```
 
### 일괄 상태 변경
```
PATCH /api/products/bulk-status
Body: { ids: string[], status: ProductStatus }
```
 
### 일괄 삭제
```
DELETE /api/products/bulk
Body: { ids: string[] }
```
 
### CSV Export
```
GET /api/products/export
Query: 목록 조회와 동일 (페이지네이션 없음)
Response: Blob (text/csv)
```
