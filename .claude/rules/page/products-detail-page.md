# 상품 상세·등록 페이지

## 0. 페이지 개요
- 경로 : `/products/new` (신규), `/products/:id` (수정)
- 목적 : 상품 생성 및 수정, 이미지 관리, 재고 관리
- 주요 사용자 : 상품 MD, 운영 담당자
- 진입 방식 : 목록의 `+ 상품 등록` / `수정` 버튼 / `상품명/코드` 텍스트 클릭

## 2. PageHeader
스크롤 시 상단 고정. 
저장 버튼 항상 접근 가능하도록 해야함.

```
┌──────────────────────────────────────────────────────────────────┐
│ ← 상품 목록   나이키 에어맥스 90          [임시저장]  [저장] [삭제]    │
└──────────────────────────────────────────────────────────────────┘
```

- 뒤로가기 : `← 상품 목록` 클릭 시 변경사항 있으면 이탈 확인 모달
- 타이틀 : 신규 - `새 상품 등록` / 수정 - 상품명 표시
- 임시저장 : `localStorage` 저장, 재진입 시 복원 안내 배너 노출
- 저장하기 : 유효성 검사 후 API 호출

### 이탈 방지
- 폼 변경 발생 시 `beforeunload` 이벤트 등록
- 뒤로가기·링크 이동 시 확인 모달:

  ```
  변경 사항을 저장하지 않고 나가시겠습니까?
  저장하지 않은 내용은 사라집니다.
  [계속 편집]  [저장하지 않고 나가기]
  ```

## 3. 기본 정보

```
┌─ 기본 정보 ──────────────────────────────────────────────────┐
│  상품명 *                                                    │
│  [────────────────────────────────────────────] 0/100자     │
│                                                             │
│  상품코드 *                              판매가 *             │
│  [───────────────────]  [중복확인]       [₩ ──────────────]  │
│  ✓ 사용 가능한 코드입니다.                                     │
└──────────────────────────────────────────────────────────────┘
```

### 3-1. 상품명(필수)
- 필드명 : `name`
- 타입 : Text Input
- 최대 길이 : 100자 (실시간 카운터 표시)
- 유효성 : 공백만 입력 불가
- 에러 메시지 : `상품명을 입력해주세요.`

### 3-2. 상품코드(필수)
- 필드명 : `productCode`
- 타입 : Text Input + 중복확인 버튼
- 형식 : 영문·숫자·하이픈 허용 (공백 불가)
- 최대 길이 : 50자 
- 자동 생성 : `신규 등록` 시 UUID prefix 기반 코드 자동 생성 제안 버튼 제공

#### 중복 확인 플로우
```
입력 → [중복확인] 클릭
  ├─ API 호출 중: 버튼 로딩 스피너
  ├─ 사용 가능: 초록 안내 문구 "사용 가능한 코드입니다."
  ├─ 중복:      빨간 에러 문구 "이미 사용 중인 상품코드입니다."
  └─ API 오류:  "확인에 실패했습니다. 다시 시도해주세요."
```
- 저장 시도 시 중복확인 미완료 상태면 저장 불가 + 안내 문구
- 코드 변경 시 확인 상태 초기화

### 3-3. 판매가(필수)
- 필드명 : `price`
- 타입 : Number Input (좌측 ₩ prefix)
- 범위 : 0 이상 양의 정수
- 포맷 : 천단위 콤마 자동 적용 (표시용)
- 에러 메시지 : `올바른 금액을 입력해주세요.`

```

│  상품 요약 설명 *                                            │
│  [──────────────────────────────────────────────] 0/200자   │
│  검색 결과 및 상품 상단에 표시되는 짧은 설명입니다.          │
│                                                              │
│  상품 간단 설명                                              │
│  [────────────────────────────────────────────────────────] │
│  상품 카드 하단에 표시되는 간략한 설명입니다. (0/500자)      │
│                                                              │
│  상품 상세 설명 *                                            │
│  ┌─[RichTextEditor]──────────────────────────────────────┐  │
│  │ B  I  U  | H1 H2 | 목록 | 링크 | 이미지 | 테이블         │  │
│  └───────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

### 4-1. 상품 요약 설명
- 타입 : Single-line Text Input
- 최대 : 200자
- 용도 : 검색 결과 미리보기, 상품 상단 노출

### 4-2. 상품 간단 설명
- 타입 : Textarea (2~3줄)
- 최대 : 500자
- 용도 : 상품 카드 하단, 목록 페이지 부가 설명

### 4-3. 상품 상세 설명
- 타입 : Rich Text Editor (WYSIWYG)
- 권장 라이브러리 : TipTap / Quill / Toast UI Editor
- 저장 형식 : HTML String

#### Rich Text Editor 지원 기능
- 텍스트 서식: Bold, Italic, Underline, Strikethrough
- 헤딩: H1 ~ H4
- 목록: Ordered / Unordered
- 링크 삽입·제거
- 이미지 삽입 (URL 입력 또는 업로드)
- 테이블 삽입 (기본 행/열)
- 정렬: Left / Center / Right
- HTML 직접 편집 모드 토글 (고급 사용자용)
- 최대 높이: 600px, 스크롤 처리

## 5. 이미지 관리
```
┌─ 이미지 관리 ────────────────────────────────────────────────┐
│  ┌─ 규격 이미지 ────────────────────────────────────────┐    │
│  │  대표 이미지 (500×500)    목록 이미지 (300×300)        │   │
│  │  ┌──────────────┐         ┌──────────────┐          │   │
│  │  │   드래그 or   │         │   드래그 or   │          │   │
│  │  │   클릭 업로드  │         │   클릭 업로드 │          │   │
│  │  └──────────────┘         └──────────────┘          │   │
│  │  작은 목록 (100×100)      축소 이미지 (220×220)        │   │
│  │  ┌──────────┐             ┌──────────────┐          │   │
│  │  └──────────┘             └──────────────┘          │   │
│  └──────────────────────────────────────────────────────┘  │
│  ┌─ 추가 이미지 (최대 20장) ────────────────────────────┐    │
│  │  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌──────────────────┐  │   │
│  │  │ 1  │ │ 2  │ │ 3  │ │ 4  │ │  + 이미지 추가     │  │   │
│  │  └────┘ └────┘ └────┘ └────┘ └──────────────────┘  │   │
│  │  드래그하여 순서 변경                                  │   │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 5-1. 이미지 규격 정책
| 타입 | `type` 값 | 최소 크기 | 권장 크기 | 비율 | 용도 |
|------|----------|----------|----------|------|------|
| 대표 이미지 | `main` | 800 × 800px | 1500 × 1500px | 1:1 | 상품 상세 대표 이미지 · 줌 기능 |
| 목록 이미지 | `list` | 400 × 400px | 600 × 600px | 1:1 | 상품 목록 그리드 썸네일 |
| 작은 목록 | `small` | 100 × 100px | 100 × 100px | 1:1 | 검색 결과 미니 썸네일 |
| 축소 이미지 | `thumbnail` | 200 × 200px | 300 × 300px | 1:1 | 장바구니·위시리스트 |
| 추가 이미지 | `extra` | 800 × 800px | 1500 × 1500px | 1:1 | 상품 상세 갤러리 (대표와 동일 권장) |

#### 공통 파일 제한
- 최대 파일 크기 : 10MB
- 허용 형식 : JPG, PNG, GIF, WebP
- 추가 이미지 최대 수 : 20장 
- 권장 파일 크기 : 300KB 이하. 페이지 로딩 속도 최적화 (압축 후 업로드 권장)
- 비율 : 1:1 (정사각형)

### 5-2. 단일 이미지 업로드 영역

#### 업로드 전 상태
```
┌────────────────────────────────┐
│      [업로드 아이콘]             │
│   클릭 또는 이미지를 끌어다        │
│   놓으세요                       │
│   JPG, PNG, GIF, WebP · 최대 10MB│
│   권장: 1500 × 1500px (1:1)     │
└────────────────────────────────┘
```

#### 업로드 후 상태
```
┌────────────────────────────────┐
│ [이미지 미리보기]                │
│  ────────────────────────────  │
│  파일명.jpg  |  2.3MB  |  [×] │
└────────────────────────────────┘
```

#### 업로드 플로우
```
파일 선택 (클릭 or Drag & Drop)
  ├─ 형식 검사 실패 → 에러 토스트 "JPG, PNG, GIF, WebP 파일만 업로드 가능합니다."
  ├─ 크기 초과 → 에러 토스트 "파일 크기는 10MB 이하여야 합니다."
  └─ 검사 통과
       ├─ 로컬 미리보기 즉시 표시 (URL.createObjectURL)
       ├─ Progress Bar 노출
       ├─ API 업로드 (Multipart)
       │   ├─ 성공 → ProductImage 객체 저장, Progress 완료
       │   └─ 실패 → 에러 토스트 + 미리보기 제거
       └─ 완료
```

#### [×] 삭제 버튼
- 클릭 시 확인 없이 즉시 제거 (단, 저장 전이면 로컬 상태만 제거)
- 이미 저장된 이미지 삭제 시 "삭제하면 복구가 불가능합니다." 확인 모달

### 5-3. 추가 이미지 영역

#### 기능 명세
- 다중 업로드 : 파일 다이얼로그에서 복수 선택 가능
- 드래그 앤 드롭 : 파일 탐색기에서 직접 드롭
- 순서 변경 : 이미지 간 드래그 앤 드롭으로 `order` 값 변경
- 삭제 : 각 이미지 우상단 [×] 버튼
- 최대 제한 : 20장 초과 시 업로드 불가 + 안내 문구

#### 추가 이미지 카드
```
┌──────────────┐
│ [미리보기]   │  ← 드래그 핸들 아이콘 (⠿) 좌상단
│              │  ← [×] 삭제 버튼 우상단
│   파일명     │
│   2.1MB      │
└──────────────┘
```

#### 업로드 중 상태
```
┌──────────────┐
│  [업로드 중]  │
│  ████░░░ 60% │
└──────────────┘
```

### 5-4. 이미지 에러 케이스
- 형식 불일치 : `JPG, PNG, GIF, WebP 파일만 업로드 가능합니다.`
- 크기 초과 : `파일 크기는 10MB 이하여야 합니다. (현재: N.NMB)`
- 추가 이미지 20장 초과 : `추가 이미지는 최대 20장까지 등록 가능합니다.`
- 업로드 실패 (서버) : `이미지 업로드에 실패했습니다. 다시 시도해주세요.`

## 6. 상품 상태
```
┌─ 상품 상태 ──────────────────────────┐
│  현재 상태                           │
│  ○ 판매중   ● 숨김   ○ 품절          │
│  [판매중]  상품이 고객에게 노출       │
│  [숨김]    고객 미노출, 재고 유지     │
│  [품절]    재고 소진 상태 표시        │
└──────────────────────────────────────┘
```
- `active` : 판매중 - 고객 노출 및 구매 가능
- `hidden` : 숨김 - 고객 미노출, 내부 관리용
- `sold_out` : 품절 - 고객 노출, 구매 불가

- Radio Button 또는 Segmented Control UI
- 상태 변경 시 즉시 반영 (저장은 전체 저장 버튼 기준)
- 자동 품절 전환 안내: `available = 0`이면 배너로 안내

  ```
  ⚠️ 가용 재고가 0입니다. 상태를 '품절'로 변경하는 것을 권장합니다.
  ```

## 7. 재고 관리
```
┌─ 재고 관리 ──────────────────────────┐
│  전체 입고    판매 수량    가용 재고  │
│    150개       105개       45개      │
│  ──────────────────────────────────  │
│  재고 조정                           │
│  ○ 입고   ● 출고                    │
│  수량                                │
│  [────────────────────]              │
│                                      │
│  사유 (선택)                         │
│  [────────────────────]              │
│                                      │
│  [재고 조정 적용]                      │
│  ──────────────────────────────────  │
│  재고 이력 보기 ↓                    │
└──────────────────────────────────────┘
```

### 7-1. 재고 현황 표시
-  전체 입고 : `stock.total` | N개 
-  판매 수량 : `stock.sold` | N개 
-  가용 재고 : `stock.available` | N개 

### 7-2. 재고 조정
- 유형 : 입고 / 출고 (Radio)
- 수량 : 양의 정수 입력 (0 불가)
- 사유 : 텍스트 입력 (선택, 최대 100자)
- 출고 제한 : 출고 수량이 `available`을 초과하면 오류

#### 입고 처리
```
stock.total += 수량
stock.available += 수량
```

#### 출고 처리
```
if (수량 > stock.available) → 에러: "가용 재고보다 많은 수량입니다."
stock.available -= 수량
// stock.total, stock.sold는 변경 없음 (실제 판매가 아니므로)
```

#### 재고 조정 완료 후
- 현황 수치 즉시 갱신
- 성공 토스트: `재고가 조정되었습니다.`

### 7-3. 재고 이력
`재고 이력 보기` 클릭 시 아코디언 or 모달로 표시.
```
┌─ 재고 이력 ───────────────────────────────────────────────────┐
│  날짜             유형    수량    사유             담당자       │
│  2024.01.15 14:30  입고    +100    초기 입고        김운영     │
│  2024.01.20 09:00  출고    -10     파손 처리        이운영     │
│  2024.01.25 11:00  입고    +60     추가 입고        김운영     │
└───────────────────────────────────────────────────────────────┘
```
- 최근 20건 기본 표시
- `더보기` 로 페이지네이션

## 9. 폼 유효성 검사

### 9-1. 필수 필드
- `name` : `상품명을 입력해주세요.` 
- `productCode` : `상품코드를 입력해주세요.` 
- `productCode` (중복확인 미완료) : `상품코드 중복 확인을 완료해주세요.`
- `price` : `판매가를 입력해주세요.`

### 9-2. 형식 검사
| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| `price` | 0 이상 정수 | `올바른 금액을 입력해주세요.` |
| `productCode` | 영문·숫자·하이픈만 허용 | `상품코드는 영문, 숫자, 하이픈만 사용 가능합니다.` |
| `name` | 최대 100자 | `상품명은 100자 이내로 입력해주세요.` |

### 9-3. 검사 시점
- 필드 Blur : 해당 필드 단독 유효성 검사
- 저장 버튼 클릭 : 전체 필드 일괄 검사
- 에러 상태 중 입력 : 실시간 에러 해제

### 9-4. 에러 표시 방식
- 필드 하단 빨간 텍스트 에러 메시지
- 필드 Border 빨간 강조
- 저장 클릭 시 첫 번째 에러 필드로 스크롤 + 포커스

## 10. 저장 플로우

### 신규 등록 (`/products/new`)
```
[저장하기] 클릭
  → 유효성 검사
     ├─ 실패: 첫 에러 필드로 스크롤 + 에러 표시
     └─ 통과
          → POST /api/products (기본정보 + 설명 + 상태)
             ├─ 성공: 생성된 상품 ID로 리다이렉트 → /products/:id
             │        성공 토스트: "상품이 등록되었습니다."
             └─ 실패: 에러 토스트, 폼 유지
```

### 수정 저장 (`/products/:id`)
```
[저장하기] 클릭
  → 유효성 검사
     └─ 통과
          → PATCH /api/products/:id
             ├─ 성공: 데이터 갱신 + 성공 토스트 "변경사항이 저장되었습니다."
             └─ 실패: 에러 토스트, 폼 유지
```

### 저장 중 UI
- 저장 버튼 로딩 스피너 + 비활성화
- 임시저장 버튼 비활성화
- 입력 필드는 유지 (수정 가능)

## 11. 임시저장
- 저장 키 : `sellops_product_draft_{userId}_{productId or 'new'}`
- 저장 시점 : 임시저장 버튼 클릭
- 저장 대상 : 이미지 URL 제외 텍스트 필드 전체
- 복원 안내 : 재진입 시 상단 배너 "임시 저장된 내용이 있습니다. [복원] [무시]"
- 만료 : 정상 저장 완료 시 삭제, 7일 후 자동 만료

## 14. 접근성
- 모든 Input에 `<label>` 또는 `aria-label` 제공
- 필수 필드 `aria-required="true"` + 시각적 `*` 표시
- 에러 상태 `aria-invalid="true"` + `aria-describedby` 에러 메시지 연결

- 드래그 앤 드롭 정렬: 키보드 대안 제공 (방향키 또는 순서 입력 필드)
- 모달 포커스 트랩 (FocusTrap)
- 색상만으로 상태 표현 금지 → 아이콘 + 텍스트 병행

## 15. 에러 처리
- 상품 조회 실패 (404) : "존재하지 않는 상품입니다." + 목록으로 이동 버튼
- 상품 조회 실패 (서버) : 에러 페이지 + 다시 시도 버튼
- 저장 실패 (충돌) : "다른 사용자가 수정했습니다. 새로고침 후 다시 시도하세요."
- 저장 실패 (서버) : 에러 토스트, 폼 유지
- 이미지 업로드 실패 : 해당 이미지 에러 상태 표시 + 재업로드 버튼
- 재고 조정 실패 : 에러 토스트, 수치 롤백
- 세션 만료 : 로그인 페이지 리다이렉트 + 임시저장 후 안내

## 16. API 연동 정의

### 상품 상세 조회
```
GET /api/products/:id
Response: ProductDetail (Product + Stock + ProductImage[])
```
 
### 상품 생성
```
POST /api/products
Body: {
  name, price, productCode,
  summary, shortDescription, description,
  status
}
Response: { id: string, ...ProductDetail }
```
 
### 상품 수정
```
PATCH /api/products/:id
Body: Partial<Product>  // 변경된 필드만
```
 
### 상품코드 중복 확인
```
GET /api/products/check-code?code={productCode}&excludeId={id}
Response: { available: boolean }
```
 
### 이미지 업로드
```
POST /api/products/:id/images
Body: FormData { file: File, type: ImageType, order?: number }
Response: ProductImage
```
 
### 이미지 삭제
```
DELETE /api/products/:id/images/:imageId
```
 
### 이미지 순서 변경
```
PATCH /api/products/:id/images/reorder
Body: { orders: Array<{ id: string, order: number }> }
```
 
### 재고 조정
```
POST /api/products/:id/stock/adjust
Body: {
  type: 'in' | 'out',
  quantity: number,
  reason?: string
}
Response: Stock
```
 
### 재고 이력
```
GET /api/products/:id/stock/history
Query: { page: number, limit: number }
Response: { items: StockHistory[], total: number }
```
