# 주문 관리 페이지 (Order Management)

## 개요

관리자가 주문을 조회, 검색, 필터링하고 상세 정보를 확인할 수 있는 페이지를 구현한다.
주요 목적은 **주문 상태 파악 및 관리 효율 향상**이다.


## 주요 기능

### 주문 리스트 조회

* 주문 목록을 테이블 형태로 표시한다.

* 컬럼 구성:

  * 주문번호
  * 고객정보 (이름, 등급)
  * 상품 정보
  * 결제 금액
  * 결제 방식
  * 배송 상태
  * 주문 일시

* 반응형:

  * Desktop: 테이블
  * Mobile: 카드형 리스트
    
### 주문 상태 필터

* 필터 옵션:

  * 전체 (default)
  * 결제 대기
  * 결제 완료
  * 배송 준비
  * 배송 중
  * 배송 완료
  * 취소
  * 환불

* 동작:

  * 상태 변경 시 페이지는 1로 초기화
  *  URL query에 반영 (`?order_status=order_confirmed`, `?payment_status=payment_completed`, `?shipping_status=shipping_in_progress`)


### 검색 기능

* 검색 대상:

  * 주문번호
  * 고객명 (한글 초성 검색 포함)
  * 이메일

* 동작:

  * debounce 300ms 적용
  * 공백 제거 후 검색 수행
  * 2글자 이상 입력 시 검색 권장 (최소 길이 제한 선택 적용 가능)

### 한글 IME 처리

* composition 이벤트 사용:

  * `compositionstart` → 검색 실행 중단
  * `compositionend` → 즉시 검색 실행
* 조합 중 중복 검색 방지

### 페이지네이션

* 기본 페이지 사이즈: N개 (상수)
* 동작:

  * 페이지 변경 시 URL query 업데이트 (`?order_page=2`)
  * 필터/검색 변경 시 페이지 1로 초기화

### URL 상태 관리

* query params:

  * `order_search`
  * `order_status`
  * `payment_status`
  * `shipping_status`
  * `order_page`

* 요구사항:

  * 새로고침 시 상태 유지
  * URL 공유 시 동일 상태 재현

### 주문 상세 보기

* 방식:

  * 상세 페이지(`/orders/:id`) 또는 모달

* 포함 정보:

  * 주문 상세 정보
  * 고객 정보
  * 상품 목록
  * 결제 정보
  * 배송 정보


### 주문 상태 변경 (선택 기능)

* 가능한 상태 흐름:

  * 결제 완료 → 배송 준비 → 배송 중 → 배송 완료

* 요구사항:

  * 사용자 액션 기반 변경 (버튼 또는 드롭다운)
  * 변경 전 confirm 필요
  * 성공 시 UI 즉시 반영


## API 스펙

### 주문 목록 조회

```http
GET /orders
```

#### Query Params

* order_status: string
* payment_status: string
* shipping_status: string
* order_search: string

#### Response

```json
{
  "items": Order[],
  "totalCount": number
}
```

### 주문 상세 조회

```http
GET /orders/:id
```

### 주문 상태 변경

```http
PATCH /orders/:id/status
```

#### Body

```json
{
  "status": "shipped"
}
```

## UX 요구사항

* 로딩 상태 표시 (스켈레톤 또는 spinner)
* 데이터 없음:

  * “검색 결과가 없습니다” 메시지 표시
* 에러 상태:

  * 재시도 UI 제공
* 모바일:

  * 카드 전체 클릭 시 상세 이동
* 테이블:

  * row hover 효과 제공


## 성능 요구사항

* 검색 debounce 적용 (300ms)
* 불필요한 리렌더 방지 (memoization)
* API 요청 중복 방지 (AbortController 또는 최신 요청만 반영)


## 예외 처리

* 검색 결과 없음
* API 실패 (네트워크 오류)
* 잘못된 query param
* 상태 변경 실패


## 확장 고려사항

* 서버 사이드 필터링 전환 가능
* 무한 스크롤로 변경 가능
* 권한별 액션 제한 (관리자/운영자)


## 완료 기준 (Definition of Done)

* 모든 기능 정상 동작
* URL 상태 유지 확인
* 모바일/데스크탑 반응형 확인
* 테스트 통과
* 타입 에러 없음

## 반응형 규칙

### PC (≥1280px) — 기본 레이아웃
- 사이드바: 고정 표시 (190px 너비)
- 주문 테이블: 모든 컬럼 표시
  - 주문번호 | 고객정보 | 상품 정보 | 결제 금액 | 결제 방식 | 배송 상태 | 주문 일시
- 테이블 푸터: 페이지네이션, 전체 레코드 수 표시

### 태블릿 (768px ~ 1279px)
- 사이드바: 아이콘만 표시 (60px), hover 시 라벨 툴팁
- 주문 테이블:
  - 일부 보조 컬럼 숨김 (예: 결제 방식)
  - 주문번호, 고객정보, 배송 상태 등 핵심 정보 표시

### 모바일 (≤767px)
- 사이드바: 완전히 숨김 → 상단 햄버거 메뉴
- 주문 테이블 → 카드형으로 변환
  - 카드 헤더: 주문번호 + 배송 상태 뱃지
  - 카드 본문: 고객 정보, 상품, 결제 금액/방식, 주문 일시
  - 2열 그리드 레이아웃으로 정보 배치

## 인터랙션 명세

- **테이블 행 hover (PC/태블릿)**: 행 배경 미세 강조
- **카드 hover (모바일)**: 카드 살짝 부상 (translateY -2px), 그림자 강조
- **클릭 동작**:
  - 전체 행/카드 클릭 시 주문 상세 페이지로 이동
  - (선택적으로) 점 3개 메뉴 버튼은 추가 액션 가능
- **상태/배송 뱃지**:
  - 색상 점 + 상태 텍스트 인라인 뱃지
  - 중요 상태는 강조 색상 (예: 취소, 환불 등)
- **페이지네이션 버튼**:
  - 활성/비활성 상태 명확히 구분
  - hover 시 배경 색 변화
- **CSV 내보내기 버튼**:
  - hover 시 강조 색상
- **검색바**:
  - 입력 시 debounce 적용 (300ms)
  - 한글 IME 입력 처리 (조합 중 검색 방지)
- **모바일 카드형**:
  - 헤더 클릭 → 상세 페이지
  - 본문 정보는 2열 스택
